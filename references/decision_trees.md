# Decision Trees

Load when the team is unsure what to do next, which test to run, or whether to scale a channel.

## What should we do next?

```text
Is there repeated evidence of urgent pain?
├── No → Run problem interviews / review mining / narrow segment.
└── Yes → Do buyers already spend money or time solving it?
    ├── No → Test trigger, pain cost, and urgency; consider lower-friction offer.
    └── Yes → Is there a reachable channel?
        ├── No → Test partnerships, communities, outbound lists, search language.
        └── Yes → Is the offer trusted and easy to buy?
            ├── No → Improve proof, pricing, risk reversal, onboarding, decision assets.
            └── Yes → Run channel/copy experiments and instrument funnel.
```

## Which test should we run?

```text
Biggest uncertainty?
├── Pain/ICP → Problem interviews, review mining, survey.
├── Switch motivation → Switch interviews, lost-deal analysis.
├── Demand → Smoke landing page, outbound, search ads.
├── Feature demand → Fake-door, concierge, prototype test.
├── Price/WTP → Pricing interview, fake pricing page, paid pilot.
├── Copy/message → Message test, 5-second test, ad angles.
├── Channel → Small-budget channel test with qualified-intent metric.
├── Activation → Onboarding test, product analytics, session review.
└── Retention → Cohort analysis, customer success interviews.
```

## Should we scale a channel?

```text
Does the channel produce ICP-qualified intent?
├── No → Do not scale.
└── Yes → Does qualified intent convert or activate?
    ├── No → Fix offer, proof, sales process, onboarding.
    └── Yes → Does CAC payback work at current price/margin?
        ├── No → Raise price, improve conversion, reduce cost, or stop.
        └── Yes → Scale gradually with guardrails.
```
