## Hero variants

1. `[Outcome] for [ICP] without [pain/risk].`
2. `The [category] built for [specific workflow/person].`
3. `Replace [alternative] with [new mechanism] in [timeframe].`
4. `Find/fix/ship [valuable thing] before [trigger].`
5. `[Product] helps [ICP] [job] by [mechanism], so they can [result].`

## Copy QA checklist

- [ ] Names the category.
- [ ] Names the buyer/segment.
- [ ] Names the painful alternative.
- [ ] Makes status quo cost visible.
- [ ] Shows mechanism, not only benefits.
- [ ] Uses buyer language from research.
- [ ] Adds proof before high-friction CTA.
- [ ] Makes price/offer understandable.
- [ ] Handles top objections.
- [ ] Uses one primary CTA.
- [ ] No unsupported claims or deceptive urgency.
